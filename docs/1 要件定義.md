# 勤怠管理アプリ 要件定義書（修正版）

## 第1章：基本情報

### 1.1 アプリ導入の目的

本システムは、現在紙で行われている勤怠管理業務を電子化し、職員の業務負担を軽減するとともに、勤怠情報の正確な把握と効率的な集計を実現することを目的とする。具体的には以下の点を改善対象とする：

- 紙媒体による出勤簿・退庁簿管理からの脱却
- 勤務時間および時間外労働の正確な記録・把握
- 時間外手当算出にかかる作業負担の軽減
- 年次有給休暇の申請・承認フローのオンライン化

### 1.2 対象ユーザー

- 常勤職員
- 臨時職員（パートタイム・フルタイム含む）
  ※委託業者や短期アルバイト職員は対象外（必要に応じて将来的に拡張可能）

### 1.3 現行運用の課題

| 項目   | 現行手法          |
|------|---------------|
| 出勤   | 出勤簿への押印       |
| 退勤   | 退庁簿への手書き記録    |
| 年休申請 | 紙様式による申請と課長決裁 |

これにより、勤怠情報の集計や手当算出に多くの時間と労力を要している。

## 第2章：機能要件

### 2.1 勤怠入力機能

- Webアプリ上で出勤・退勤ボタンを押下し、システム時刻を自動で記録する。
- 打刻時刻は改ざん防止のため自動記録のみ。
- 今後、GPSやIP制限などによる打刻制限の実装も検討。

### 2.2 休暇等申請機能（MVP対象）

| 申請種別 | 備考         |
|------|------------|
| 有給休暇 | 全職員対象      |
| 代休   | 振替休日の申請として |
| 特別休暇 | 忌引き等の申請    |

その他の申請（欠勤、遅刻等）は将来的に追加可能。

### 2.3 申請承認フロー

- 原則、課長が承認者。
- 課長職の場合、副町長が承認者。
- 承認者は1名。ただし代理承認者の設定を検討する（例：課長不在時の対応）。
- 申請→確認→承認・却下の単一ステップ。

### 2.4 勤怠集計・タイムシート

- 月別の勤怠情報を一覧表示・集計。
- 時間外勤務は以下の条件に基づき補正率を適用：

| 補正条件            | 補正率の例 |
|-----------------|-------|
| 平日・所定時間外        | 1.25  |
| 土日祝日            | 1.35  |
| 深夜（22:00〜翌5:00） | 1.5   |

- タイムシートはCSV／PDFで出力可能。
- 補正率はDBや管理画面から変更可能な構成とする。

#### 【補正率の根拠】

- 時間外勤務手当の支給に関する条例および規則に基づき、補正率は以下のように定義される：
    - 平日所定時間外勤務：125%
    - 土日祝日勤務：135%
    - 午後10時〜翌朝5時の深夜勤務：上記に25%を加算（例：150% → 175%）
    - 再任用短時間職員等が8時間未満の勤務をした場合：100%
    - 月60時間超過分：150%（深夜の場合は175%）
    - 代替措置が指定された場合などは減額の可能性あり（最低125%）
- 詳細な適用基準については、町の時間外勤務手当支給条例第11条および関連規則第24条、第24条の2〜3に準拠する。

### 2.5 管理者機能（人事課）

- 全職員の勤怠閲覧
- アカウント管理（新規登録、削除）
- 所属部署情報の編集
- 補正率や休暇種別の管理
- 役職の変更は職員自身が可能（申請ベースの承認機構も将来的に検討）

## 第3章：技術要件・運用

### 3.1 利用端末・環境

- 主に庁内PCからの利用を想定
- レスポンシブ対応（スマホ、タブレット対応）必須
- インターネット経由でのアクセスに対応
- 将来的にネイティブアプリ（iOS/Android）対応も検討

### 3.2 ログイン認証

- ID・パスワードによるログイン認証
- 初期設定：初期パスワードはランダム生成し、メール通知等で共有する
- 初回ログイン時にパスワード変更を必須とする

### 3.3 セキュリティ要件

- HTTPS対応（SSL/TLS）
- パスワードはハッシュ化（例：bcrypt）
- セッション有効期限の設定（自動ログアウト）
- アクセス権限による画面・データ制御
- ログイン／打刻履歴の保存（監査ログ）
- タイムゾーンは明示的に「+09:00」とする

## 第4章：その他要件

### 4.1 特殊勤務形態への対応

- 宿直：日をまたぐ時間管理への対応
- 出張：勤務場所に関する申請機能の追加を将来的に検討

### 4.2 外部連携

- 人事評価や給与システムとの連携は現時点では実施しない

### 4.3 スケジュール

- 明確なリリース時期は設定せず、段階的な開発・運用を想定

### 4.4 最小構成（MVP）

- 出退勤打刻
- 年休等申請
- 承認フロー（単一）
- 勤怠集計・補正率反映
- アカウント管理機能
