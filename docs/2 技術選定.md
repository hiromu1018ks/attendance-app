# 勤怠管理アプリ 技術選定書

## 1. 目的と背景

本技術選定書は、役場職員の勤怠管理アプリを開発・運用するにあたり、使用する技術スタックおよびその選定理由を明確にすることを目的とする。

アプリケーションはAWS上に構築し、フロントエンドにはReact、バックエンドにはJava（Spring Boot）を用いてリッチかつ安定したWebシステムの構築を目指す。

## 2. アーキテクチャ全体像

```
[React Frontend] -- API通信 --> [Spring Boot Backend] --JPA--> [PostgreSQL DB]
                                                │
                                                └--> [AWS上にデプロイ（EC2）]
```

## 3. 技術選定の方針（選定基準）

- モダンなユーザーインターフェースを実現できること
- Java/Spring Bootによる堅牢なバックエンド構築が可能であること
- クラウド（AWS）上での運用を前提とし、将来の拡張性にも対応できること
- GitHubによるバージョン管理、CI/CDによる自動化に対応できること
- 運用担当者がIT初心者でも扱いやすい環境を整備できること

## 5. 各技術の採用理由と比較検討

### React（フロントエンド）
- 利用実績が豊富で、学習資源も多い
- モダンで柔軟なUIが実装可能
- Tailwind CSSとの相性も良く、迅速なUI構築が可能

### Spring Boot（バックエンド）
- Javaベースの堅牢なフレームワーク
- セキュリティやバリデーション、DB接続などの機能が充実
- REST API構築との親和性が高い

### PostgreSQL（データベース）
- オープンソースで高機能なRDBMS
- 時系列データや複雑な集計に強い
- AWS RDSとしても安定した運用が可能

### AWS（インフラ）
- スケーラブルで柔軟な構成が可能
- 初期はEC2による簡易構成で導入し、将来的に必要に応じてECS/Fargateへの移行も可能
- RDSでDBの運用管理も自動化できる

### GitHub + GitHub Actions（開発フロー）
- 無料プランでもCI/CD導入が可能
- チーム開発や履歴管理に強み
- Dockerと連携してデプロイの自動化も容易

## 6. 拡張性・保守性に関する考慮

- API設計はRESTベースで設計し、将来的なモバイルアプリ化に対応
- 権限管理（一般職員／課長／人事課）などはロールベースで実装予定
- フロントエンドはコンポーネント分割により再利用性と可読性を重視
- ログや監査証跡はクラウドログ管理（CloudWatch等）に対応予定
- IT初心者でも操作できるよう、簡易UIと操作マニュアルを整備予定
